# TapTap 十周年（第一期）H5 Demo 需求说明（Demo 级）

> 目标：用一个可交互的 H5 Demo，把 4 月“十周年开启”版本的核心体验讲清楚（不追求真实数据、也不需要接后端）。

## 1. 4月版本的三块核心内容

### A. 发福利 & 账号数据回顾
- **玩法**：用户进入“十周年回顾”页面，看到自己的 TapTap 游戏经历/数据回顾，并可生成分享卡。
- **领奖逻辑**：回顾页展示的“数据维度”会对应若干福利的**解锁条件**；满足条件可领取。
- **绑定推动（关键）**：
  - 并非强制所有人绑定。
  - 如果用户当前不满足某些福利条件，允许其通过“新增绑定数据”来**补齐条件并领取**（让用户感到绑定有用）。

### B. 发现好游戏 & 零分成
- **内容展陈**：专题页展示“陪伴 TapTap 一路成长的游戏/开发者故事/平台坚持”，用于新用户认知教育。
- **TapMaker GameJam 试玩场地**：
  - 试玩是活动内容本身（浏览试玩列表、进入试玩详情、模拟试玩完成）。
  - 参与试玩/反馈可获得活动积分。
  - 在叙事上植入“发现好游戏”“零分成”信息；并提到“前几名获得发行支持”（Demo 只需文案表达，不做真实报名系统）。

### C. 活动积分商店
- **兑换内容**：
  - 头像框 / 徽章（可在个人资料展示）
  - 积分抽点券：**小概率中奖，大概率落空**

## 2. Demo 要表达的四个目标（体验上）
- **目标1（福利 + 数据价值）**：用户在领福利时，能明确感知“我在 TapTap 的经历/数据是有用的”，从而更愿意绑定。
- **目标2（发现好游戏/零分成认知）**：用户能在专题和试玩里快速理解 TapTap 的差异化主张。
- **目标3（参与贡献）**：用户不止领取福利，也能通过“试玩/反馈/互动”产出内容或行为，获取积分。
- **目标4（纪念痕迹）**：活跃参与者能获得可长期展示/对外分享的纪念资产（头像框/徽章/分享卡）。

## 3. Demo 交互范围（页面清单）

### P0 必做（第一期 Demo 最小闭环）
1) **活动首页**
   - 三大模块入口：回顾 & 福利 / 好游戏专题 & 试玩 / 积分商店
   - 展示：当前积分、已领取福利数、已获得头像框/徽章
2) **十周年回顾页（个人回顾）**
   - 展示若干“回顾数据卡片”（Demo 用假数据）
   - 对应“福利卡片”列表：已解锁/未解锁、领取按钮
   - 未解锁福利提供“去绑定数据”入口（弹窗模拟绑定）
   - **分享**：生成一张“十周年回顾分享卡”（可下载图片或模拟复制链接）

#### 回顾数据卡片（生涯维度）
> 说明：本段为回顾页“生涯数据卡片”的口径与清单（不做“今年”限定）。数据为 0 的卡片不展示。

#### TapTap 生涯数据卡片库（用于扩充为“很多卡片”）
> 目标：不仅展示统计数据，也要能产出“纪念/故事感”的回顾点；默认按生涯口径统计。
>
> 你已确认的口径：
> - **最爱玩游戏**：按**总时长**（可统计范围内）计算 Top1/Top3
> - **深夜纪念**：两张卡——深夜玩游戏、深夜社区活跃
> - **开发者维度**：进入默认生涯数据卡片（为 0 不展示）

##### 0) 通用口径与展示规则
- **口径**：生涯累计 / 历史最常见 / 历史最佳（不做“今年”限定）
- **展示规则**：为 0 的卡片不展示（避免空卡）；可展示即有内容与可领奖（按活动设计）
- **敏感信息**：时间只用“时段”（如 0–3 点）表达；社交不点名、不过度曝光个人内容

##### A) 基础数据（注册 / 活跃）
1) **注册时间 + 活跃天数（合并一张）**
   - 展示：`在 {regDate} 注册，在 TapTap 活跃了 {daysActive} 天`
3) **最长连续活跃**
   - 展示：`最长连续活跃 {streakMax} 天`
4) **在 TapTap 消费了多少钱**
   - 展示：`在 TapTap 消费 {spendTotal}`

##### B) 玩游戏维度（时长 / 偏好 / 成就 / 排行）
1) **总游玩时长（生涯）**
   - 展示：`总游玩时长 {playTimeTotal}`
2) **最爱玩的 Top3 游戏（合并一张，按总时长）**
   - 展示：`最爱玩：{topGame1}/{topGame2}/{topGame3}`
   - 备注：Top1 的口径为“总时长最高”
3) **最爱的游戏类型 + 三个标签（合并一张）**
   - 展示：`最爱类型：{favoriteGenre} · 标签：{tag1}/{tag2}/{tag3}`
4) **最喜欢在什么平台玩游戏（改名）**
   - 展示：`最喜欢在 {platformPreference} 玩游戏`（手游/PC/云玩…）
6) **成就与里程碑（系列，可多张）**
   - 展示：`累计解锁成就 {achievementsTotal}` / `最难得成就：{rareAchievement}`
7) **小众宝藏（纪念/分享向）**
   - 展示：`你玩过最小众的一款：{nicheGame}`
8) **上过多少个排行榜（改口径）**
   - 展示：`你上过 {leaderboardsCount} 个排行榜`
9) **深夜玩游戏（纪念卡，合并到本维度）**
   - 展示：`深夜 {nightSlot} 你最常玩的游戏：{nightTopGame}`
10) **第一次里程碑（并入本维度，可多张）**
   - 第一次玩到通关 / 第一次解锁稀有成就 / 第一次上榜（如有）

##### C) 社区活跃维度（评价 / 论坛 / 互动 / 分享）
1) **游戏评价（改名为维度标题）**
   - 卡片A：写过的评价数量：`{reviewsCount}`
   - 卡片B：评价获得的点赞数：`{reviewLikesTotal}`
   - 卡片C：被点赞最多的评价：`{topLikedReviewTitle}`（或摘要）`· {topLikedReviewLikes} 赞`
2) **论坛与社区**
   - 发帖：`{postsCount}` / 回帖：`{repliesCount}`
   - 点赞：`{likesCount}` / 收藏：`{favoritesCount}` / 分享：`{sharesCount}`
   - 最常逛的版块/圈子：`{topForum1}/{topForum2}/{topForum3}`
3) **被点赞最多的帖子（替换“一句金句”）**
   - 展示：`{topLikedPostTitle} · {topLikedPostLikes} 赞`
4) **和好友发了多少条消息（替换好友模块）**
   - 展示：`你和好友共发送 {friendMessagesCount} 条消息`
5) **深夜社区活跃（纪念卡，合并到本维度）**
   - 展示：`深夜 {nightSlot} 你仍在社区：{nightTopAction}`
6) **第一次里程碑（并入本维度，可多张）**
   - 第一次发帖 / 第一次回帖 / 第一次帖子被点赞（如有）

##### D) 开发者维度（默认卡组，0 不展示）
1) **玩家是多少游戏的开发者（改口径）**
   - 展示：`你是 {devGamesCount} 个游戏的开发者`
2) **认证明细（每个认证单独统计）**
   - 例如：`{plannerCertCount} 个游戏策划认证 / {programmerCertCount} 个游戏程序认证 / …`（可扩展更多角色）
3) **GameJam / 聚光灯参与（认证单独统计）**
   - 参与次数：`{gamejamCount}` / 作品数：`{gamejamWorks}`
4) **创意关卡创作者（认证单独统计）**
   - 关卡数：`{levelsCount}` / 被游玩：`{levelsPlays}` / 被收藏：`{levelsFavs}`
5) **TapTap 制造创作者（认证单独统计）**
   - 作品数：`{creatorWorks}` / 被使用：`{creatorUses}` / 被点赞：`{creatorLikes}`
6) **创作高光（纪念点，合并到本维度，可多张）**
   - 第一次发布作品 / 第一次被推荐 / 第一次登上活动页（如有事件标记）
7) **“第一次被看见”（合并到本维度）**
   - 展示：`第一次被标“有帮助”的评测：{firstHelpfulDateMonth}`（如更适合也可放到社区维度）

#### 福利领取（并入回顾卡片，不单独做“福利解锁”列表）
> 说明：每张回顾卡片下方直接展示“达成条件 + 可领取奖励 + 领取按钮”；按钮状态：未达成 / 可领取 / 已领取。

##### A) 生涯数据（入场冻结）
- 进入活动后**停止累计**，以用户进入活动当下的账号数据为准
- **数据为 0 的卡片不展示**
- **展示出来的卡片都可直接领奖**（不存在“未达成”状态）
- 奖励为“按数据值计算”的 Demo 逻辑（可在此调整口径与奖励曲线）

##### B) 可补齐数据（活动期间新增）
- 进入活动后仍可通过**绑定**来新增数据并领奖（例如绑定 Steam、绑定游戏数据）
- 会存在“未绑定 / 可领取 / 已领取”的状态流转

**卡片（仅两张）**
- **绑定 Steam 账号**：绑定后一次性领奖
- **绑定更多游戏角色**：按绑定角色数量领奖；每绑定 1 个角色即可领取一次奖励（可持续绑定）
3) **好游戏专题 & TapMaker GameJam 试玩（在首页内展示）**
   - 展示：平台坚持的三句话（发现好游戏/零分成/评分公正）
   - “陪伴成长的游戏”列表（卡片式，可弹窗展开）
   - 试玩游戏列表 → 详情 → “完成试玩/提交一句反馈”
   - 完成后发放积分（+X），并提示可去积分商店兑换
5) **活动积分商店**
   - 兑换头像框/徽章：消耗积分，获得后收藏为纪念资产（本期 Demo 不做个人资料展示）
   - 积分抽点券：抽奖动画/结果（大概率空）

### P1 可选增强（有余力再做）
- 回顾页“时间轴式里程碑”（2016-2026）
- “福利分层”更清晰：新手/回归/深度用户不同档位

## 4. Demo 需要覆盖的关键状态（方便评审时演示）
- **用户类型切换（Demo 开关）**
  - 新用户：数据少、可通过绑定补齐部分福利
  - 老用户：数据多、可直接解锁更多福利
- **是否已绑定外部/游戏数据**
  - 未绑定：部分福利不可领 → 引导绑定
  - 已绑定：立即刷新条件与可领状态
- **积分变化**
  - 试玩完成 + 积分入账
  - 商店兑换/抽奖扣积分
  - 资产入库后在“我的”页可展示

## 5. 文案/视觉基调（Demo 级）
- 主题：**十年同行**（保持初心、创新突破）
- 语气：真诚、克制、强调“坚持与公正”
- 视觉：移动端大卡片、轻动效；统一用“十周年”色系（Demo 可用占位色）

- 真实点券发放、真实抽奖概率与风控
