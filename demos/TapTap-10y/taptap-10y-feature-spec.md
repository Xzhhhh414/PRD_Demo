## 模块一：入口与开场
### 入口
- 用户通过活动链接（任意 URL）进入
### 正常流程
1. 若用户**首次进入**或**未登录**，展示开场页
2. 开场页展示：品牌标题、协议勾选项（**默认未勾选**）、进入按钮
3. 用户主动勾选协议后，点击进入按钮
4. 若用户未登录，手机端唤起客户端进行登录，Web 端弹出扫码登录页面，登录成功后继续
5. 进入活动首页
### 状态与规则
| 条件              | 表现                          |
| --------------- | --------------------------- |
| 若首次进入或未登录       | 则展示开场页，登录并通过后标记为已通过         |
| 若协议未勾选，用户点击进入按钮 | 则 toast 提示需先同意协议，不进入 |
| 若点击协议链接文字       | 则弹窗展示活动协议全文                 |
| 若已登录且非首次进入      | 则跳过开场页，直接进入活动首页             |

> 活动首页及所有功能模块均需登录后方可访问，未登录用户不会看到活动首页内容。

## 模块二：活动首页
首页从上到下依次包含：
1. **顶部栏**：品牌标题 + 返回按钮（首页隐藏）
2. **置顶区**：名片缩略图入口 + 纪念币展示 + 福利兑换商店 + 每日签到
3. **十年回顾模块**：回顾卡片横向滚动列表
4. **发现好游戏模块**：猜游戏翻牌卡片列表
5. **游戏游乐场模块**：初体验打卡 + 游戏分页轮播
6. **相关活动模块**：Banner 列表
### 置顶区内容
进入首页时，置顶区会展示名片缩略图（包括当前背景、角色、贴纸的缩略预览）、纪念币数值、福利兑换商店和每日签到入口。
### 置顶区显示/隐藏
| 条件         | 表现                 |
| ---------- | ------------------ |
| 进入页面时      | 默认显示               |
| 用户开始滚动页面   | 隐藏置顶区              |
| 用户停止滚动     | 显示置顶区              |
| 页面位于最顶部    | 始终显示               |
### 每日签到
置顶区包含每日签到卡片，展示当前签到天数、连续签到天数、签到按钮。
| 条件          | 表现                                       |
| ----------- | ---------------------------------------- |
| 若今日未签到      | 显示签到按钮（高亮可点击），点击后领取纪念币 |
| 若今日已签到      | 签到按钮置灰（disabled）                     |
| 若连续签到达到指定天数 | 签到纪念币翻倍，卡片展示翻倍状态                          |

连续签到判定：若上次签到日期为昨天，则连签天数 +1；否则连签重置为 1。
### 导航
- 点击置顶区名片缩略图或名片编辑链接 → 打开名片编辑弹窗
- 点击福利兑换商店 → 打开十周年福利弹窗

**数据规则说明入口**
卡片区域顶部的数据截止日期旁有"!"图标，点击弹出弹窗，展示取数口径说明和各卡片的领奖次数规则。

**空数据的卡片展示**
若某张卡片的数据条件为0，卡片仍然存在但核心数据替换为一句鼓励性的空态文案。

**卡片展示规则**
| 条件 | 表现 |
| --- | --- |
| 有待领取的奖励 | 展示卡片内容 + 领取按钮 |
| 奖励已全部领取 | 展示卡片内容，按钮显示为已领取状态（disabled） |

**排序规则**
- 按卡片定义顺序展示，不做动态重排

**卡片滚动交互**
- 进入首页时，自动聚焦到排序最靠前的有奖励可领取的卡片位置
- 领取奖励后，若当前卡片奖励已领完，自动滚动到下一张可领取的卡片

**奖励领取**
每张卡片可领取纪念币奖励，每张卡片的单次纪念币数量和最大领取次数见上方卡片列表。部分卡片支持多次领取，按钮上显示剩余可领次数（如"×3次"），并提示"XX可提升领奖次数"。

**领取流程**
1. 用户点击领取按钮
2. 若本次领取触发了点券红包，**先弹出红包弹窗**，展示红包金额，提供"开心收下"和"查看钱包"两个操作
3. 红包弹窗关闭后，弹出纪念币弹窗，展示本次获得的纪念币数量及可兑换奖品预览
4. 纪念币弹窗提供两个操作：
   - 继续领取奖励
   - 跳转到福利兑换商店
   - 弹窗支持勾选"不再提示"，后续领取跳过弹窗直接发放纪念币（但若触发红包，红包弹窗仍会弹出）
5. 若该卡片奖励已全部领完，关闭后卡片标记为已领取
6. 当前卡片奖励全部领完后，自动滚动到下一张可领取的卡片

**点券红包**

领取纪念币时，可能额外触发点券红包奖励。红包金额在一定范围内之间随机，最小精度 0.01 元，点券直接发放至用户账户余额（钱包）。

**触发规则**
| 条件 | 规则 |
| --- | --- |
| 空数据卡片（数据为 0 的生涯卡） | 仅可领取 1 次纪念币，**不触发**红包 |
| 有数据的卡片（生涯卡、Steam 卡、角色数据卡） | 每次领取纪念币均触发红包 |

**红包总量限制**
- 活动设有点券红包的**全局总数量上限**，所有用户共享
- 红包库存耗尽后，不再触发红包，仅正常发放纪念币
- 运营可在活动期间追加红包数量，追加后立即生效

**后台管理**
- 权限 key：`点券红包`
- 用签到系统作为后台

| 条件 | 表现 |
| --- | --- |
| 若本次领取触发红包且库存有余 | 先弹红包弹窗，关闭后再弹纪念币弹窗 |
| 若勾选了"不再提示"但触发红包 | 红包弹窗仍弹出，纪念币弹窗跳过 |
| 若红包库存为 0 | 正常领取纪念币，不触发红包 |
| 若为空数据卡片 | 不触发红包 |

### 绑定Steam卡片
| 条件 | 表现 |
| --- | --- |
| 若未绑定 | 展示空白卡片 + 绑定引导描述 + 绑定按钮 |
| 若已绑定 | 展示 Steam 数据卡片（挚爱佳作、账号价值、佳作数量、游玩时长）+ 领取按钮 |
| 若已绑定但游戏数为 0 | 展示空态描述 |
| 若已领取 | 按钮变为已领取状态 |

绑定流程：点击绑定按钮 → 跳转 Steam 绑定页面 → 完成绑定后自动返回活动页 → 卡片展示 Steam 数据 → 可领取纪念币（仅限 1 次）

### 绑定游戏角色（战绩）
每绑定一个游戏角色，生成一张独立的战绩卡片，展示角色信息（头像、名称、所属游戏、职业/等级、战绩数据）。
**首次进入活动时的默认展示：**
- 若玩家已有绑定过战绩的游戏角色，默认展示所有已绑定战绩的卡片，每张卡片需**手动领取**纪念币
- 若玩家未绑定过任何游戏角色战绩，展示一张空白引导卡片，点击可前往绑定页面

**进入活动后新绑定：**
- 通过卡片区域右上角绑定入口可继续绑定更多角色
- 新绑定的角色会在回顾区域生成新的战绩卡片
- 新绑定产生的纪念币需**手动领取**，绑定完成后自动滚动至新卡片，玩家点击领取按钮领奖

绑定流程：点击绑定按钮 → 跳转战绩绑定页面 → 完成绑定后自动返回活动页 → 卡片展示战绩数据 → 自动滚动至新卡片 → 玩家手动领取纪念币（每个角色卡片能领 1 次）

**排序：** 未领取的角色卡排在已领取的前面。

### 分享十年回顾
**入口**
- 回顾区域上方的分享按钮 → 打开分享弹窗

**分享卡片内容**
分享卡片汇总展示用户的生涯数据摘要，包括：
- 用户昵称、ID
- 数据列表（按数据可用性动态展示，每行 4 项，末行不满 4 项时居中排列）：相伴时光、游玩游戏数、游玩时长、深夜探索次数、预约新作数、累计消费、喜爱类型、游戏成就数、我的挚爱、独家宝藏数、编辑之选数、玩家评价数、发布内容数、获赞数、深夜冲浪天数、徽章收藏数、好友数、粉丝数、游戏创作数
- 生涯数据卡片可领取纪念币总数，不算绑steam和角色数据卡片
- 活动介绍的文本

**分享弹窗**
- 展示分享卡片
- 展示分享链接二维码，扫码是进入活动首页
- 提供分享和下载图片按钮
- 分享至：优先调用系统分享功能
- 下载图片：生成海报图片并下载

**分享海报**
下载图片时生成海报，内容与分享卡片一致（用户昵称/ID、数据列表、可领取纪念币总数、活动介绍文本），额外附带活动二维码和链接。

## 模块四：发现好游戏
### 正常流程
1. 游戏来源：提前定好的若个游戏卡片，我给到开发。大概是6-10个。默认是未揭示的状态。
2. 未揭示的游戏显示为**神秘卡**（横向布局）：问号图标 + 提示语 + 故事按钮 + 纪念币预览
3. 用户点击神秘卡 → 揭示游戏 → 自动打开游戏故事弹窗
4. 游戏故事弹窗为**分页图文**：每页是图片 + 一段故事文字，底部圆点指示器 + 翻页按钮，最后一页为「关闭」按钮
5. 关闭故事弹窗后，**自动领取纪念币**（首次）
6. 已揭示的卡片展示：游戏图标 + 游戏名 + 标签 + 热门评论跑马灯 + 重听故事按钮 + 留言板按钮
7. 点击已揭示卡片的游戏图标或游戏名 → 跳转至该游戏的详情页
### 留言板
- 点击留言板按钮 → 直接跳转至提前创建的帖子详情页，用户可在帖子下评论盖楼
### 热门评论跑马灯
- 已揭示的游戏卡片展示热门评论跑马灯
- 评论来源为该游戏对应帖子中的玩家回复，按点赞数降序取 Top 5
- 跑马灯逐条垂直滚动，每条展示点赞数和评论内容
- 若无热门评论，不展示跑马灯区域
### 状态与规则
| 条件           | 表现                    |
| ------------ | --------------------- |
| 若游戏未揭示       | 则显示神秘卡               |
| 若游戏已揭示但未听完故事 | 则显示游戏信息 + 故事按钮       |
| 若游戏已揭示且已领取纪念币 | 则显示游戏信息 + 重听故事按钮     |
| 翻牌后          | 不可回到未揭示状态             |
| 每款游戏纪念币       | 仅可领取一次，关闭故事弹窗时自动发放    |

## 模块五：游戏游乐场
### 游戏列表
- 游戏来源：提前定好的若干游戏卡片，由运营提供。会挑 标题旁的「游戏来源：TapTap制造」可点击跳转至 TapTap 制造页面
- 游戏列表以**分页轮播**展示，每页展示6个，所有游戏都会出现在列表中
- 每款游戏卡片展示：游戏图标、游戏名称、热度值、标签列表
- 已打卡的游戏卡片展示「已打卡」标记
- 卡片下方提供跳转按钮，点击可前往游戏页面

**排序规则**
整体排序优先级：**未打卡的游戏排在前面，已打卡的游戏排在后面**。同一分组内按热度降序排列。
第一页与后续页采用不同的排序逻辑：
| 页面 | 排序方式 |
|---|---|
| **第一页** | 从未打卡的游戏中进行分层抽取 + 去重轮换（见下方随机规则），抽取完成后随机打乱顺序。若未打卡游戏不足一页，用已打卡游戏补位 |
| **第二页及之后** | 将第一页未选中的剩余游戏按未打卡优先 + 热度降序排列，依次分页展示 |

有「换一换」功能，随机刷新第一页游戏列表，其余页可横划查看。每次进入和刷新活动页面自动执行一次随机

**第一页随机规则**
采用**分层抽取 + 去重轮换**机制，确保每次刷新既有热门游戏也有新锐游戏，且与上一次展示尽可能不重复：
1. **分层**：将未打卡的游戏按热度排序，前 50% 归入「热门组」，后 50% 归入「新锐组」
2. **抽取**：每次从热门组和新锐组各随机抽取等量游戏组成第一页
3. **去重**：记录上一次展示的游戏 ID 列表，本次抽取时优先从上次**未展示**的游戏中选取
4. **补位**：若某组中排除上次已展示后剩余不足，则先选完剩余的，再从上次已展示的该组游戏中随机补足
### 初体验打卡
统计玩家在活动期间玩过的游乐场游戏数量（游戏时长 > 0 即算打卡），每打卡一款游戏获得 50 纪念币。
活动从4月17日开始，截止时间会先配置到今年年底。
**打卡规则**
- 玩家在活动期间玩过的游乐场游戏，只要该游戏的游玩时长大于 0，即视为完成该游戏的打卡
- 每款游戏只计一次打卡
- 每次进入活动时自动更新打卡数量并领取纪念币
**任务展示**
模块顶部（标题下方、游戏列表上方）展示打卡信息：
1. **奖励说明**：每体验一款新游戏，自动获得纪念币
2. **当前打卡数**：已打卡 X 款

**初始状态**
- 首次进入时，打卡数为 0，进度条为空

## 模块六：十周年名片
### 入口
- 活动首页顶部置顶区：名片缩略图 / 名片编辑链接 → 打开名片编辑弹窗
### 名片内容
名片展示以下信息：
- **角色形象**：各个游戏角色的头像，默认角色为嗒啦啦（tarara01），支持拖拽移动和缩放
- **背景主题**：带有图案/纹理的渐变色背景
- **贴纸**：用户添加的 emoji 贴纸，支持拖拽移动和缩放
- **个人信息**：昵称、ID（展示在名片卡片下方同一行，不在卡片内部）
### DIY 编辑
编辑弹窗顶部为名片实时预览区，下方为三个 Tab 切换面板。Tab 顺序为：角色 → 背景 → 贴纸，默认展示角色 Tab。
每次编辑自动保存，没有手动保存按钮。
**角色 Tab（默认展示）**
- 展示所有角色选项（包括品牌形象角色、游戏角色等），默认选中嗒啦啦（tarara01）
- 互斥选择，选中即应用
- **所有角色默认拥有，无需纪念币兑换**，点击即可直接切换
- 角色在名片上支持拖拽调整位置

**背景 Tab**
- 展示所有背景主题选项
- 互斥选择，选中即应用
- 默认解锁 1 种背景，其余需消耗纪念币解锁
- 解锁后可自由切换

**贴纸 Tab**
- 展示所有贴纸选项
- 可多选，无数量上限
- 同种贴纸在名片上只能使用一次
- 点击已使用的贴纸 → 从名片上移除
- 点击未使用的贴纸 → 添加到名片（默认位置为右上区域）
- 默认解锁 1 种基础贴纸，其余需消耗纪念币解锁
- 已添加的贴纸支持拖拽调整位置，位置数据持久化
**缩放与拖拽交互**
- 角色和贴纸均支持拖拽移动和缩放
- 点击名片上的角色或贴纸后选中，底部出现缩放滑块控件
- 缩放范围：最小 20%，最大 200%
- 拖动滑块实时调整缩放比例
- 拖拽范围大于名片可见区域，可将角色/贴纸的一部分拖出名片边界实现遮挡效果
- 名片区域设有裁切，超出部分自动隐藏
- 再次点击已选中的角色或贴纸可取消选中
- 点击名片空白区域取消选中，缩放滑块隐藏
**解锁流程（仅适用于背景、贴纸）**
实现方式：Admin兑换
1. 用户点击未解锁的背景/贴纸选项
2. 弹出确认弹窗：展示名称 + 消耗纪念币 + 当前纪念币余额
3. 若纪念币足够 → 点击确认 → 扣除纪念币 → 解锁 → toast 提示已解锁
4. 若纪念币不足 → 弹窗提示纪念币不足

### 保存为图片
名片编辑弹窗提供保存为图片按钮，点击后打开保存图片弹窗：
1. 实时渲染名片预览，角色和贴纸的位置、大小、裁切效果与名片编辑中一致
2. 可勾选显示内容（背景、贴纸、角色）
3. 可选裁剪形状（方形/圆形）
4. 方形支持独立调节宽度和高度；圆形支持调节直径
5. 裁剪框支持拖拽调整中心点
6. 点击保存按钮后，按裁剪区域导出图片并下载

### 分享名片
名片编辑弹窗提供分享按钮，点击后打开分享弹窗：
- 展示当前名片预览（只读，贴纸不可拖拽）
- 展示分享链接二维码
- 提供分享和下载图片按钮
- 分享弹窗提供返回按钮，点击后关闭分享弹窗并重新打开名片编辑弹窗
- 分享弹窗关闭后停留在当前页面，不跳转
分享行为：
- **分享至**：优先调用系统分享功能
- **下载图片**：生成名片海报图片并下载
- 若用户通过名片分享链接进入，则直接进入活动首页。

### 首次分享领取十周年徽章
用户首次点击名片分享按钮时，自动获得「Tap十周年徽章」。

**引导提示**（未获得徽章时展示）
- 活动首页置顶区名片入口上方，浮动气泡提示"领取十周年徽章"
- 名片编辑弹窗的分享按钮上方，浮动气泡提示"分享领取十周年徽章"

**领取流程**
1. 用户点击分享按钮，打开分享弹窗
2. 分享弹窗中展示徽章领取通知，告知用户已获得徽章
3. 领取后气泡提示消失，后续分享不再触发

| 条件 | 表现 |
| --- | --- |
| 若未获得徽章 | 名片入口和分享按钮上显示气泡引导 |
| 若首次点击分享 | 打开分享弹窗并展示徽章领取通知 |
| 若已获得徽章 | 不显示气泡引导，分享行为正常 |

## 模块七：十周年福利（纪念币商店）
### 入口
- 活动首页置顶区福利兑换商店 → 打开十周年福利弹窗
- 回顾卡片领取纪念币的弹窗中提供入口 → 打开十周年福利弹窗
弹窗顶部展示当前纪念币余额。
### 一、纪念币抽奖
实现方式：Admin抽奖
每个奖品有独立的库存上限，抽完即止。
**抽奖流程**
1. 每次参与消耗纪念币，不限次数
2. 点击抽奖按钮 → 落定到中奖奖品
3. 弹出中奖结果弹窗，展示奖品名称与图标
4. 若未中奖（所有奖品库存耗尽），弹窗展示未中奖提示
5. 所有中奖记录自动记入「我的奖品」。

**状态规则**
| 条件          | 表现                          |
| ----------- | --------------------------- |
| 若纪念币充足       | 则抽奖按钮可点击                    |
| 若纪念币不足       | 则抽奖按钮 disabled              |
| 若奖池所有奖品均已抽空 | 则抽奖按钮 disabled              |
| 奖池中已抽完的单个奖品 | 置灰标记                        |

### 二、纪念币兑换
兑换区位于抽奖区下方，展示可兑换商品，包括 **Tap头像框和徽章** 和 **游戏礼包码** 两类。

**兑换流程**
1. 用户点击商品的「兑换」按钮
2. 弹出确认弹窗：商品名 + 消耗纪念币
3. 若纪念币足够 → 确认后扣除纪念币：
   - **头像框/徽章**：商品进入用户资产，弹出兑换成功弹窗，提示使用方式
   - **游戏礼包码**：弹出兑换成功弹窗，展示礼包码内容及兑换方式说明
4. 若纪念币不足 → 弹窗提示纪念币不足
5. 所有兑换记录自动记入「我的奖品」
### 三、我的奖品
- 位置：福利页面的上方，以可折叠抽屉形式展示
- 始终显示（无论是否有奖品记录）
- 默认收起，点击标题栏可展开/收起
- 若尚无任何奖品记录，展开后显示空态引导文案
- 展开后以列表形式展示 **所有抽奖中奖记录 + 兑换记录**，每条记录包含：奖品图标、奖品名称、来源标签（抽奖/兑换）、获奖时间
- 按时间倒序排列（最新在最前）
- 不同来源和类别的奖品在列表中展示额外信息：

### 四、奖品类型和展示
| 奖品类别  | 列表中的额外内容        |
| ----- | --------------- |
| 头像框   | 提示使用方式          |
| 徽章   | 提示使用方式          |
| 优惠券   | 提示兑换路径          |
| 云玩时长  | 不是兑换码，直接到账的          |
| 游戏礼包码  | 展示兑换码内容，提示兑换方式     |
