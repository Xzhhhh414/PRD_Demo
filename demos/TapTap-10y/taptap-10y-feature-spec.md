## 模块一：入口与开场

### 入口

- 用户通过活动链接（任意 URL）进入

### 正常流程

1. 若用户**首次进入**或**未登录**，展示开场页
2. 开场页展示：品牌标题、协议勾选项（**默认未勾选**）、进入按钮
3. 用户主动勾选协议后，点击进入按钮
4. 若用户未登录，手机端唤起客户端进行登录，Web 端弹出扫码登录页面，登录成功后继续
5. 进入活动首页


### 状态与规则


| 条件              | 表现                          |
| --------------- | --------------------------- |
| 若首次进入或未登录       | 则展示开场页，登录并通过后标记为已通过         |
| 若已登录且非首次进入      | 则跳过开场页，直接进入活动首页             |
| 若协议未勾选，用户点击进入按钮 | 则 toast 提示需先同意协议，不进入 |
| 若点击协议链接文字       | 则弹窗展示活动协议全文                 |

## 模块二：活动首页

### 入口

- 通过开场页进入
- 再次打开活动直接进入
- 从其他子页面点击「返回」进入

### 结构

首页从上到下依次包含：

1. **顶部栏**：品牌标题 + 返回按钮（首页隐藏）
2. **置顶区（Sticky Stats）**：名片缩略图入口 + 名片编辑链接 + 纪念币展示 + 签到卡片 + 十周年福利入口
3. **十年回顾模块**：回顾卡片横向滚动列表
4. **发现好游戏模块**：猜游戏翻牌卡片列表
5. **游戏游乐场模块**：游玩时长任务 + 游戏分页轮播
6. **相关活动模块**：Banner 列表

### 签到

置顶区包含签到卡片，展示当前签到天数、连续签到天数、签到按钮。


| 条件          | 表现                                       |
| ----------- | ---------------------------------------- |
| 若今日未签到      | 显示签到按钮（高亮可点击），点击后领取纪念币 |
| 若今日已签到      | 签到按钮置灰（disabled）                     |
| 若连续签到达到指定天数 | 签到纪念币翻倍，卡片展示翻倍状态                          |


连续签到判定：若上次签到日期为昨天，则连签天数 +1；否则连签重置为 1。

### 置顶区折叠/展开

置顶区支持自动折叠与手动展开：


| 条件           | 表现                                       |
| ------------ | ---------------------------------------- |
| 进入页面时        | 默认展开，展示完整内容（名片缩略图、纪念币卡片、签到卡片）             |
| 用户向下滚动超过一定距离 | 自动收起为精简模式，仅显示：小名片头像 + 纪念币数值 + 签到天数 + 展开按钮 |
| 用户点击展开按钮     | 恢复为完整展开状态                                |
| 手动展开后，用户再次滚动 | 再次自动收起                                   |
| 用户滚动回到页面顶部   | 自动恢复展开状态                                 |


手动展开后短暂忽略滚动事件，避免立即收起。

### 状态与规则


进入首页时，置顶区会展示名片缩略图（包括当前背景、角色、贴纸的缩略预览）、纪念币数值、签到入口和十周年福利入口。


### 导航

- 点击置顶区名片缩略图或名片编辑链接 → 打开名片编辑弹窗
- 点击十周年福利入口 → 打开十周年福利弹窗

#### 卡片展示规则

| 条件 | 表现 |
| --- | --- |
| 若卡片有数据 | 展示对应内容 + 领取按钮 |
| 若卡片无数据 | 展示空态文案 + 占位插画，仍可领取保底纪念币 |
| 若卡片奖励已全部领取 | 按钮显示为已领取状态（disabled） |

#### 排序规则

- 默认排序：按卡片定义顺序展示
- 排序优化：未领取的卡片排在已领取的前面，组内保持原序

### 卡片滚动交互

- 横向滚动，每张卡片吸附对齐
- 支持桌面端鼠标拖拽滚动
- 底部圆点分页指示器，点击圆点可跳转到对应卡片
- 进入首页时，自动聚焦到最早可领取的卡片位置
- 领取奖励后，自动滚动到下一张可领取的卡片

### 奖励领取

每张卡片可领取纪念币奖励，部分卡片支持**多次领取**：

1. 用户点击领取按钮
2. 随机发放纪念币，增加到用户余额
3. 若该卡片支持多次领取且尚未达到上限，按钮恢复为可领取状态
4. 若达到最大领取次数，按钮变为已领取状态

#### 多次领取规则

每张卡片的最大领取次数由该维度的数据量决定，数据越丰富可领次数越多。例如：

| 卡片 | 领取次数规则示例 |
| --- | --- |
| 相伴时光 | 按注册年数，每年可领 1 次 |
| 冒险旅程 | 按游玩游戏数，每 50 款可多领 1 次（上限 10） |
| 游玩时光 | 按游玩时长，每 200 小时可多领 1 次（上限 8） |
| 游戏成就 | 按成就总数 + 白金数（上限 15） |
| 玩家之声 | 按评价数 + 嘴替数（上限 15） |
| 生涯挚爱 | 固定 1 次 |

### 绑定类卡片

#### 绑定 Steam

| 条件 | 表现 |
| --- | --- |
| 若未绑定 | 展示空白卡片 + 绑定引导描述 + 绑定按钮 |
| 若已绑定 | 展示 Steam 数据卡片（挚爱佳作、账号价值、佳作数量、游玩时长）+ 领取按钮 |
| 若已绑定但游戏数为 0 | 展示空态描述 |
| 若已领取 | 按钮变为已领取状态 |

绑定流程：点击绑定按钮 → 弹出绑定弹窗 → 确认绑定 → 卡片展示 Steam 数据 → 可领取纪念币（仅限 1 次）

#### 绑定游戏角色（战绩卡片）

每绑定一个游戏角色，生成一张独立的战绩卡片，展示角色信息（头像、名称、所属游戏、职业/等级、战绩数据）。

**首次进入活动时的默认展示：**

- 若玩家已有绑定过战绩的游戏角色，默认展示所有已绑定战绩的卡片，每张卡片需**手动领取**纪念币
- 若玩家未绑定过任何游戏角色战绩，展示一张空白引导卡片，点击可前往绑定页面

**进入活动后新绑定：**

- 通过卡片区域右上角绑定入口可继续绑定更多角色
- 新绑定的角色会在回顾区域生成新的战绩卡片
- 新绑定产生的纪念币**自动发放**，无需手动领奖

**排序：** 未领取的角色卡排在已领取的前面。

## 模块四：发现好游戏（Discover Games）

### 入口

- 活动首页发现好游戏区域（位于回顾区域下方）

### 正常流程

1. 以列表形式展示若干「猜游戏」卡片
2. 未揭示的游戏显示为**神秘卡**（横向布局）：问号图标 + 提示语 + 故事按钮 + 纪念币预览
3. 用户点击神秘卡 → 揭示游戏 → 自动打开游戏故事弹窗
4. 游戏故事弹窗为**分页图文**：每页是图片 + 一段故事文字，底部圆点指示器 + 翻页按钮，最后一页为「关闭」按钮
5. 关闭故事弹窗后，**自动领取纪念币**（首次）
6. 已揭示的卡片展示：游戏图标 + 游戏名 + 标签 + 热门评论跑马灯 + 重听故事按钮 + 留言板按钮

### 留言板

- 点击留言板按钮 → 打开弹窗，引导用户前往提前创建的帖子详情页评论盖楼

### 热门评论跑马灯

- 已揭示的游戏卡片展示热门评论跑马灯
- 评论来源为预置的用户留言数据，按点赞数排序取前若干条
- 跑马灯逐条垂直滚动，每条展示点赞数和评论内容
- 若无热门评论，静态展示空态引导文案


### 状态与规则


| 条件           | 表现                    |
| ------------ | --------------------- |
| 若游戏未揭示       | 则显示神秘卡               |
| 若游戏已揭示但未听完故事 | 则显示游戏信息 + 故事按钮       |
| 若游戏已揭示且已领取纪念币 | 则显示游戏信息 + 重听故事按钮     |
| 翻牌后          | 不可回到未揭示状态             |
| 每款游戏纪念币       | 仅可领取一次，关闭故事弹窗时自动发放    |

## 模块五：游戏游乐场（Game Playground）

### 入口

- 活动首页游戏游乐场区域（位于发现好游戏下方）

### 游戏列表

- 游戏列表以**分页轮播**展示，每页展示多款游戏（2 列网格），底部圆点指示器
- 每款游戏卡片展示：游戏图标（取游戏名首字）、游戏名称、热度值、标签列表
- 卡片下方提供跳转按钮，点击可前往游戏页面
- 支持随机刷新第一页游戏列表，其余游戏可横划查看
- 每次刷新页面自动执行一次随机

#### 随机规则

采用**分层抽取 + 去重轮换**机制，确保每次刷新既有热门游戏也有新锐游戏，且与上一次展示尽可能不重复：

1. **分层**：将全部游戏按热度排序，前 50% 归入「热门组」，后 50% 归入「新锐组」
2. **抽取**：每次从热门组和新锐组各随机抽取等量游戏组成第一页
3. **去重**：记录上一次展示的游戏 ID 列表，本次抽取时优先从上次**未展示**的游戏中选取
4. **补位**：若某组中排除上次已展示后剩余不足，则先选完剩余的，再从上次已展示的该组游戏中随机补足

该机制保证：

- 每次刷新热门与新锐游戏各占一半
- 连续两次刷新之间重复游戏数量最少
- 仅记录上一次展示列表即可，无需全局历史

### 游玩时长任务

不再按单个游戏发放纪念币，改为统计玩家在所有游乐场游戏中的**累计游玩时长**，按档位自动领取纪念币奖励。共分为多个档位，时长从分钟级到小时级递增，对应纪念币奖励逐档递增。

#### 任务展示

模块顶部（标题下方、游戏列表上方）展示当前阶段的任务进度：

1. **鼓励文案** + 右侧为换一换按钮
2. **当前目标**：展示当前档位的目标时长和对应纪念币
3. **进度条**：当前累计时长 / 目标时长（使用与目标相同的时间单位：分钟或小时）
4. **进度条状态行**：可点击，点击展开/收起全部任务列表

#### 全部任务列表

- 展开后展示所有档位
- 排序规则：已达成未领取排最前 → 未完成次之 → 已领取排最后
- 每个档位显示：目标时长、纪念币奖励、当前状态（进度/已达成/已领取）
- 已领取的档位置灰显示

#### 自动领奖

- 当玩家累计游玩时长达到某档位要求时，**自动发放**对应纪念币，无需手动领取
- 一次时长增长可能同时解锁多个档位，纪念币累加发放

#### 初始状态

- 首次进入时，显示最低档位的任务进度

### 边界与异常

- 游玩时长由服务端统计，活动页加载/刷新时拉取最新时长
- 每个档位纪念币仅发放一次

## 模块六：十周年名片

### 入口

- 活动首页顶部置顶区：名片缩略图 / 名片编辑链接 → 打开名片编辑弹窗
- 分享链接 → 进入后自动打开名片分享弹窗

### 名片内容

名片展示以下信息：

- **背景主题**：带有图案/纹理的渐变色背景
- **贴纸**：用户添加的 emoji 贴纸（位置、缩放、旋转均可自定义）
- **角色形象**：用户选择的角色图片/emoji
- **品牌标识**
- **已装备装饰**：若已装备头像框/徽章则展示对应标签
- **个人信息**：昵称、ID

### DIY 编辑

编辑弹窗顶部为名片实时预览区，下方为三个 Tab 切换面板。Tab 顺序为：角色 → 背景 → 贴纸，默认展示角色 Tab。

#### 角色 Tab（默认展示）

- 展示所有角色选项（包括 Tap 头像、品牌形象角色、游戏角色等）
- 互斥选择，选中即应用
- **所有角色默认拥有，无需纪念币兑换**，点击即可直接切换
- 角色类型包括：图片角色（展示实际图片）、游戏角色（展示占位图标）

#### 背景 Tab

- 展示所有背景主题选项（多种渐变色主题）
- 互斥选择，选中即应用
- 默认解锁 1 种基础背景，其余需消耗纪念币解锁
- 解锁后可自由切换

#### 贴纸 Tab

- 展示所有贴纸选项
- 可多选，上限 10 个
- 同种贴纸在名片上只能使用一次
- 点击已使用的贴纸 → 从名片上移除
- 点击未使用的贴纸 → 添加到名片（默认位置为右上区域）
- 默认解锁 1 种基础贴纸，其余需消耗纪念币解锁
- 已添加的贴纸支持拖拽调整位置，位置数据持久化


| 条件           | 表现                         |
| ------------ | -------------------------- |
| 若贴纸已达上限，尝试添加 | 则 toast 提示已达上限 |


#### 解锁流程（仅适用于背景、贴纸）

> 角色全部免费拥有，点击即可直接切换，无需解锁。

1. 用户点击未解锁的背景/贴纸选项
2. 弹出确认弹窗：展示名称 + 消耗纪念币 + 当前纪念币余额
3. 若纪念币足够 → 点击确认 → 扣除纪念币 → 解锁并立即应用 → toast 提示已解锁
4. 若纪念币不足 → 弹窗提示纪念币不足

### 贴纸拖拽交互

- 拖拽范围限制在名片区域内
- 点击贴纸可切换为「选中」状态
- 拖拽过程中实时更新贴纸位置
- 拖拽结束后保存位置数据

### 保存为图片

名片编辑弹窗 header 提供保存为图片按钮，点击后打开保存图片弹窗：

1. 实时渲染名片预览
2. 可勾选显示内容（背景、贴纸、角色）
3. 可选裁剪形状（方形/圆形）
4. 方形支持独立调节宽度和高度
5. 圆形支持调节直径
6. 裁剪框支持拖拽调整位置
7. 点击保存按钮 → 按裁剪区域导出图片并下载

### 分享名片

名片编辑弹窗 header 提供分享按钮，点击后打开分享弹窗：

- 展示当前名片预览（只读，贴纸不可拖拽）
- 展示分享链接二维码
- 提供分享和下载图片按钮
- 分享弹窗 header 提供返回按钮 → 关闭分享弹窗，重新打开名片编辑弹窗
- 分享弹窗关闭后停留在当前页面，不跳转

分享行为：

- **分享至**：优先调用系统分享功能；若不支持则复制链接至剪贴板，并 toast 提示已复制
- **下载图片**：生成名片海报图片并下载

### 分享链接进入


| 条件                | 表现                                         |
| ----------------- | ------------------------------------------ |
| 若通过名片分享链接进入 | 先进入首页，然后自动打开名片分享弹窗（展示当前用户的名片），关闭后回到首页 |

## 模块七：十周年福利（纪念币商店）

### 入口

- 活动首页置顶区十周年福利入口 → 打开十周年福利弹窗
- 通过深链进入 → 跳转首页后自动打开十周年福利弹窗

弹窗顶部展示当前纪念币余额。

### 一、福利抽奖

#### 奖池

奖池包含 **5 个类别**，每个类别 2 个奖品：


| 类别    | 说明                            |
| ----- | ----------------------------- |
| 头像框   | 获得后可在个人主页装备使用                 |
| 点券    | 点券发放至账户余额，可购买站内游戏、CDKey、云玩服务等 |
| 优惠券   | 购买游戏时可用，有使用门槛和有效期             |
| 云玩时长  | 获得云玩兑换码，前往兑换中心兑换时长            |
| CDKey | 发放具体兑换码，前往兑换中心使用              |


每个奖品有独立的库存上限，抽完即止。

#### 抽奖流程

1. 每次参与消耗纪念币，不限次数
2. 点击抽奖按钮 → 落定到中奖奖品
3. 弹出中奖结果弹窗，展示奖品名称与图标
4. 若未中奖（所有奖品库存耗尽），弹窗展示未中奖提示

#### 中奖结果弹窗（按类别差异化）


| 奖品类别  | 弹窗内容                          |
| ----- | ----------------------------- |
| 头像框   | 提示使用方式                        |
| 点券    | 说明用途，底部增加查看钱包按钮               |
| 优惠券   | 展示兑换路径，提醒尽快兑换                 |
| 云玩时长  | 提示获得兑换码，展示兑换路径                |
| CDKey | 直接展示兑换码内容，引导前往兑换中心使用          |


所有弹窗均提供关闭按钮，不展示消耗纪念币信息。

#### 状态规则


| 条件          | 表现                          |
| ----------- | --------------------------- |
| 若纪念币充足       | 则抽奖按钮可点击                    |
| 若纪念币不足       | 则抽奖按钮 disabled              |
| 若奖池所有奖品均已抽空 | 则抽奖按钮 disabled              |
| 奖池中已抽完的单个奖品 | 置灰标记                        |


### 二、我的奖品

- 位置：十周年福利弹窗内**最顶部**（纪念币显示下方、抽奖区上方），以可折叠抽屉形式展示
- 始终显示（无论是否有奖品记录）
- 默认收起，点击标题栏可展开/收起
- 若尚无任何奖品记录，展开后显示空态引导文案
- 展开后以列表形式展示 **所有抽奖中奖记录 + 兑换记录**，每条记录包含：奖品图标、奖品名称、来源标签（抽奖/兑换）、获奖时间
- 按时间倒序排列（最新在最前）
- 不同来源和类别的奖品在列表中展示额外信息：

**抽奖奖品：**

| 奖品类别  | 列表中的额外内容        |
| ----- | --------------- |
| 头像框   | 提示使用方式          |
| 点券    | 名称右侧显示查看钱包按钮   |
| 优惠券   | 提示兑换路径          |
| 云玩时长  | 提示兑换路径          |
| CDKey | 提示兑换路径          |

**兑换奖品：**

| 奖品类型   | 列表中的额外内容   |
| ------ | ---------- |
| 游戏礼包码  | 提示兑换方式     |
| 其他兑换商品 | 无额外提示      |

### 三、纪念币兑换

兑换区位于抽奖区下方（两区域间有分割线），展示可兑换商品，包括 **纪念装饰（头像框、徽章）** 和 **游戏礼包码** 两类。

#### 商品类型

| 类型     | 说明                                       |
| ------ | ---------------------------------------- |
| 头像框/徽章 | 装饰类商品，兑换后进入用户资产，可装备展示                    |
| 游戏礼包码  | 兑换后展示礼包码（key）及兑换方式说明；卡片上显示已兑换及对应码值 |

#### 兑换流程

1. 用户点击商品的「兑换」按钮
2. 弹出确认弹窗：商品名 + 消耗纪念币 + 当前余额
3. 若纪念币足够 → 确认后扣除纪念币：
   - **装饰类**：商品进入用户资产 → 自动装备 → toast 提示已兑换
   - **游戏礼包码**：弹出兑换成功弹窗，展示礼包码内容及兑换方式说明
4. 若纪念币不足 → 弹窗提示纪念币不足
5. 所有兑换记录自动记入「我的奖品」

#### 装备规则（仅限装饰类商品）


| 条件         | 表现                     |
| ---------- | ---------------------- |
| 若商品未拥有     | 则显示兑换按钮（标注所需纪念币）        |
| 若商品已拥有且已装备 | 则显示已装备状态（非高亮）          |
| 若商品已拥有但未装备 | 则显示装备按钮（高亮），点击切换装备状态   |


装备后效果：

- 头像框/徽章标签展示在名片顶部右侧
- 名片 DIY 弹窗中同样展示对应装备按钮
