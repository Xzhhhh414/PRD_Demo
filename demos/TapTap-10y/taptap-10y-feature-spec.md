> 本文档面向开发与测试，描述各功能模块的产品行为。不涉及产品背景与战略，仅描述「做什么」和「怎么表现」。
> 不包含每张回顾卡片使用哪些数据的部分（已单独维护）。

## 全局约定

### 登录态

| 条件 | 表现 |
|------|------|
| 若用户未登录 | 始终停留在开场页（「开启我的 TapTap 十年回顾」），点击进入按钮后弹出登录弹窗，登录成功后方可进入活动 |
| 若用户已登录且已通过开场页 | 再次打开活动直接进入活动首页，无需重复开场流程 |

> 活动首页及所有功能模块均需登录后方可访问，未登录用户不会看到活动首页内容。

### 积分体系

积分为活动内唯一通用货币。

#### 获取途径

| 获取途径 | 说明 |
|----------|------|
| 每日签到 | 每天签到可领取积分；连续签到达到指定天数后，每日签到奖励翻倍 |
| 十年回顾卡片领奖 | 每张有数据的卡片均可领取一次，积分额度由卡片配置决定 |
| 绑定 Steam 账号 | 绑定后可领取积分（一次性） |
| 绑定游戏角色 | 每绑定 1 个角色可领取积分，可多次绑定 |
| 发现好游戏 | 翻牌揭示游戏并听完故事后自动领取积分（每款游戏一次） |
| 游戏游乐场时长任务 | 累计游玩时长达标自动领取，多档累进奖励 |

#### 消耗途径

| 消耗途径 | 说明 |
|----------|------|
| 名片 DIY 解锁装饰 | 背景、贴纸需消耗积分解锁（角色全部免费拥有） |
| 积分兑换 | 头像框、徽章需消耗积分兑换 |
| 福利抽奖 | 消耗积分参与，每日 1 次 |

### 数据快照机制

用户进入活动时，系统冻结其生涯数据为「进入时快照」（`careerSnapshot`），后续活动外的行为不影响回顾展示与发奖计算。若用户切换测试数据源，快照重新生成。

### 奖励领取飞入动效

领取积分后，展示飞行动画：

1. 从触发位置生成 chip 元素
2. chip 沿弧线轨迹飞向顶部积分 pill 展示区
3. 到达时目标 pill 展示脉冲反馈动画
4. 若用户启用了 `prefers-reduced-motion`，跳过飞行动画，仅短暂显示后移除

### 分享通用逻辑

所有分享入口遵循统一策略：

1. **分享至**：优先调用 `navigator.share`（系统/应用内分享）；若不支持则复制链接至剪贴板，并 toast 提示「已复制链接，可粘贴分享」
2. **下载图片**：生成包含活动内容和二维码的 SVG 海报 → Canvas 绘制 → 导出 PNG 并触发下载
3. 分享弹窗关闭后停留在当前页面，不跳转

### 无障碍

- 所有交互元素提供 `aria-label`
- 弹窗使用 `role="dialog"` + `aria-modal="true"`
- toast 使用 `role="status"` + `aria-live="polite"`
- 若用户系统启用 `prefers-reduced-motion: reduce`，所有动画降级为无动画或最小化过渡

### Toast 提示

全局 toast 组件，展示操作反馈信息，自动消失。同一时间仅展示一条。

## 模块一：入口与开场

### 入口

- 用户通过活动链接（任意 URL）进入

### 正常流程

1. 若用户**首次进入**或**未登录**，展示开场页
2. 开场页展示：品牌标题（「TapTap 十周年」）、视觉动画舞台区、协议勾选项、「开启我的 TapTap 十年回顾」按钮
3. 用户勾选协议后，点击按钮
4. 若用户未登录，弹出登录弹窗，登录成功后继续
5. 开场页播放退出过渡动画后，进入活动首页

### 状态与规则

| 条件 | 表现 |
|------|------|
| 若首次进入或未登录 | 则展示开场页，登录并通过后标记为已通过 |
| 若已登录且非首次进入 | 则跳过开场页，直接进入活动首页 |
| 若协议未勾选，用户点击进入按钮 | 则 toast 提示「请先阅读并同意活动协议」，不进入 |
| 若点击协议链接文字 | 则弹窗展示活动协议全文 |

### 动画

开场页包含以下动画层：

- 光环呼吸
- 圆环脉冲
- 光点漂移（3 个光点）
- 品牌进入动画
- 退出过渡：开场页淡出 + 活动主体淡入

### 边界与异常

- 若 URL 带 `?reset=1`，清空本地状态后重新加载（回到首次进入状态）
- 若 URL 带 `?persist=0`，每次加载均使用默认状态（不持久化）

## 模块二：活动首页

### 入口

- 通过开场页进入
- 再次打开活动直接进入
- 从其他子页面点击「返回」进入

### 结构

首页从上到下依次包含：

1. **顶部栏**：品牌标题 + 返回按钮（首页隐藏）
2. **置顶区（Sticky Stats）**：名片缩略图入口 + 「编辑十周年名片」链接 + 积分 pill + 签到卡片 + 十周年福利入口
3. **十年回顾模块**：回顾卡片横向滚动列表 + 分享按钮
4. **发现好游戏模块**：猜游戏翻牌卡片列表
5. **游戏游乐场模块**：游玩时长任务 + 游戏分页轮播
6. **相关活动模块**：Banner 列表

### 签到

置顶区包含签到卡片，展示当前签到天数、连续签到天数、签到按钮。

| 条件 | 表现 |
|------|------|
| 若今日未签到 | 显示「立即签到」按钮（高亮），点击后领取积分 + 飞行动画 + toast 提示 |
| 若今日已签到 | 显示「今日已签」按钮（disabled） |
| 若连续签到达到指定天数 | 签到积分翻倍，卡片展示翻倍状态 |

连续签到判定：若上次签到日期为昨天，则连签天数 +1；否则连签重置为 1。

### 置顶区折叠/展开

置顶区支持自动折叠与手动展开：

| 条件 | 表现 |
|------|------|
| 进入页面时 | 默认展开，展示完整内容（名片缩略图、积分卡片、签到卡片） |
| 用户向下滚动超过一定距离 | 自动收起为精简模式，仅显示：小名片头像 + 积分数值 + 签到天数 + 展开按钮 |
| 用户点击展开按钮 | 恢复为完整展开状态 |
| 手动展开后，用户再次滚动 | 再次自动收起 |
| 用户滚动回到页面顶部 | 自动恢复展开状态 |

收起态与展开态之间使用过渡动画平滑切换。手动展开后短暂忽略滚动事件，避免立即收起。

### 状态与规则

| 条件 | 表现 |
|------|------|
| 进入首页 | 置顶区展示名片缩略图（当前背景+角色+贴纸的小预览）、积分数值、签到、十周年福利入口 |
| 若路由为 `#/discover`、`#/recap`、`#/shop` | 则重定向到首页并滚动到对应模块 / 打开对应弹窗 |

### 导航

- 点击置顶区名片缩略图或「编辑十周年名片」→ 打开名片编辑弹窗
- 点击「十周年福利」入口 → 打开十周年福利弹窗
- 在非首页页面，顶部栏显示「返回」按钮 → 返回首页

## 模块三：十年回顾

### 入口

- 活动首页十年回顾模块区域
- 首次回顾流程（从开场结束后进入，当前默认关闭，直接进首页）

### 卡片展示规则

回顾数据按维度拆分为多张卡片（「回顾卡片」），分为两组轮播：

1. **生涯数据卡片组**：基于玩家生涯数据生成的各维度卡片
2. **绑定奖励卡片组**：绑定 Steam、绑定角色等需要额外操作的卡片

卡片展示规则：

| 条件 | 表现 |
|------|------|
| 若某维度数据为 0 或无数据 | 则该卡片不展示 |
| 若卡片有数据 | 则展示对应内容和「领取」按钮 |
| 若卡片奖励已领取 | 则按钮显示为「已领取」（disabled），不可再次领取 |
| 若卡片存在 alias（合并卡） | 则旧 ID 领过后，新合并 ID 的剩余奖励自动扣减已领部分 |

### 卡片滚动交互

- 横向滚动 + scroll-snap，每张卡片吸附对齐
- 支持桌面端鼠标拖拽滚动
- 底部圆点分页指示器，点击圆点可跳转到对应卡片
- 进入首页时，自动聚焦到最早可领取的卡片位置（仅首次，避免领奖后跳动）
- 领取奖励后，飞行动画结束后自动滚动到下一张可领取的卡片

### 奖励领取

1. 用户点击「领取」按钮
2. 标记该卡片为已领取
3. 增加对应积分到用户余额
4. 播放飞行动画
5. 刷新页面状态，按钮变为「已领取」

### 绑定类卡片

| 类型 | 行为 |
|------|------|
| 绑定 Steam | 未绑定时显示「绑定 Steam」按钮 → 打开绑定弹窗（展示 Steam 账号信息预览）→ 确认绑定 → 成功后可领取积分，仅可领取一次 |
| 绑定游戏角色（首次，回顾卡片内） | 未绑定时显示「绑定角色」按钮 → 打开绑定弹窗（展示可绑定角色卡片轮播）→ 确认绑定第一个角色 → 回到卡片后需**手动点击「领取」**领取积分 |
| 绑定游戏角色（后续，右上角入口） | 通过卡片右上角「+ 绑定游戏角色」入口继续绑定更多角色 → 每绑定 1 个角色后**自动发放**积分，无需手动领奖 |

### 首次回顾流程（可选，当前默认关闭）

若启用首次回顾流程：

1. 开场结束后，进入首次回顾页面（独立路由 `#/firstrecap`）
2. 顶部隐藏常规导航栏，仅展示品牌标题 + 积分数值
3. 依次展示回顾卡片，逐张解锁浏览
4. 每张卡片可独立领取奖励
5. 生涯数据卡片浏览完后，若有可领取的绑定卡片，继续展示
6. 全部完成后，弹出「完成」弹窗，汇总本次获得的积分总额（仅展示一次）
7. 页面底部提供「先进入活动会场，稍后再回顾」跳过入口，点击直接进入首页

### 分享十年回顾

回顾区域提供「分享」按钮，点击后打开分享弹窗：

- 展示用户昵称、ID
- 展示回顾摘要卡片（包含关键词标签、挚爱游戏、游戏数量、时长等摘要信息）
- 展示分享链接二维码
- 提供「分享至」和「下载图片」按钮
- 下载的海报图包含：品牌信息、玩家关键词、挚爱游戏列表、年度挚爱、互相陪伴天数、二维码

### 边界与异常

- 若进入活动时尚未生成快照，自动冻结数据
- 重复点击已领取的卡片无响应

## 模块四：发现好游戏（Discover Games）

### 入口

- 活动首页「发现好游戏」区域（位于回顾区域下方）
- 标题下方换行显示英文副标题
- 标题右侧注明平台主张（如「TapTap 坚持：零分成」）

### 正常流程

1. 以列表形式展示若干「猜游戏」卡片
2. 未揭示的游戏显示为**神秘卡**（横向布局）：问号图标 + 提示语 + 「听听它的故事」按钮 + 积分预览
3. 用户点击神秘卡 → 播放翻牌动画 → 揭示游戏 → 自动打开游戏故事弹窗
4. 游戏故事弹窗为**分页图文**：每页一个 emoji + 一段故事文字，底部圆点指示器 + 翻页按钮，最后一页为「关闭」按钮
5. 关闭故事弹窗后，**自动领取积分**（首次），toast 提示 + 飞行动画
6. 已揭示的卡片展示：游戏图标 + 游戏名 + 标签 + 热门评论跑马灯 + 「再听听它的故事」按钮 + 「留言板」按钮
7. 点击已揭示的卡片（非按钮区域）→ 弹出游戏详情说明弹窗

### 热门评论跑马灯

- 已揭示的游戏卡片展示热门评论跑马灯
- 评论来源为预置的用户留言数据，按点赞数排序取前若干条
- 跑马灯逐条垂直滚动，每条展示格式：`👍 {点赞数} {评论内容}`
- 若无热门评论，静态展示「还没有热评，快来留言做第一个上墙的人吧」
- 若用户启用 `prefers-reduced-motion`，跑马灯不自动滚动

### 留言板

- 点击「留言板」按钮 → 打开弹窗，引导用户前往提前创建的帖子详情页评论盖楼

### 状态与规则

| 条件 | 表现 |
|------|------|
| 若游戏未揭示 | 则显示神秘卡（问号 + 提示语） |
| 若游戏已揭示但未听完故事 | 则显示游戏信息 + 「听听它的故事」按钮 |
| 若游戏已揭示且已领取积分 | 则显示游戏信息 + 「再听听它的故事」按钮 |
| 翻牌后 | 不可回到未揭示状态 |
| 每款游戏积分 | 仅可领取一次，关闭故事弹窗时自动发放 |

## 模块五：游戏游乐场（Game Playground）

### 入口

- 活动首页「游戏游乐场」区域（位于发现好游戏下方）
- 标题下方换行显示英文副标题「Game Playground」
- 标题右侧注明游戏来源：TapTap制造 聚光灯GameJam

### 游戏列表

- 游戏列表以**分页轮播**展示，每页展示多款游戏（2 列网格），底部圆点指示器
- 每款游戏卡片展示：游戏图标（取游戏名首字）、游戏名称、热度值、标签列表
- 点击卡片弹出游戏详情说明弹窗（Demo 中不实际跳转，展示跳转平台和目标 URL）
- 支持「🎲 换一换」随机刷新第一页游戏列表，其余游戏可横划查看
- 每次刷新页面自动执行一次随机

#### 随机规则

采用**分层抽取 + 去重轮换**机制，确保每次刷新既有热门游戏也有新锐游戏，且与上一次展示尽可能不重复：

1. **分层**：将全部游戏按热度排序，前 50% 归入「热门组」，后 50% 归入「新锐组」
2. **抽取**：每次从热门组和新锐组各随机抽取等量游戏组成第一页
3. **去重**：记录上一次展示的游戏 ID 列表，本次抽取时优先从上次**未展示**的游戏中选取
4. **补位**：若某组中排除上次已展示后剩余不足，则先选完剩余的，再从上次已展示的该组游戏中随机补足

该机制保证：
- 每次刷新热门与新锐游戏各占一半
- 连续两次刷新之间重复游戏数量最少
- 仅记录上一次展示列表即可，无需全局历史

### 游玩时长任务

不再按单个游戏发放积分，改为统计玩家在所有游乐场游戏中的**累计游玩时长**，按档位自动领取积分奖励。共分为多个档位，时长从分钟级到小时级递增，对应积分奖励逐档递增。

#### 任务展示

模块顶部（标题下方、游戏列表上方）展示当前阶段的任务进度：

1. **鼓励文案**：「去游乐场里寻找自己喜欢的游戏吧~」，右侧为「🎲 换一换」按钮
2. **当前目标**：「活动期间累计游玩 X，自动领取 Y 积分」
3. **进度条**：当前累计时长 / 目标时长（使用与目标相同的时间单位：分钟或小时）
4. **进度条状态行**：可点击，点击展开/收起全部任务列表

#### 全部任务列表

- 展开后展示所有档位
- 排序规则：已达成未领取排最前 → 未完成次之 → 已领取排最后
- 每个档位显示：目标时长、积分奖励、当前状态（进度/已达成/已领取）
- 已领取的档位置灰显示
- 列表底部包含测试工具（仅 demo 使用）：增加游玩时长选择器 + 增加/重置按钮

#### 自动领奖

- 当玩家累计游玩时长达到某档位要求时，**自动发放**对应积分，无需手动领取
- toast 提示「自动领取 N 积分」
- 一次时长增长可能同时解锁多个档位，积分累加发放

#### 初始状态

- 首次进入时，显示最低档位的任务进度

### 边界与异常

- 游玩时长由服务端统计，活动页加载/刷新时拉取最新时长
- 每个档位积分仅发放一次

## 模块六：十周年名片

### 入口

- 活动首页顶部置顶区：名片缩略图 / 「编辑十周年名片」链接 → 打开名片编辑弹窗
- 分享链接 `#/sharememorial` → 进入后自动打开名片分享弹窗

### 名片内容

名片展示以下信息：

- **背景主题**：带有图案/纹理的渐变色背景
- **贴纸**：用户添加的 emoji 贴纸（位置、缩放、旋转均可自定义）
- **角色形象**：用户选择的角色图片/emoji
- **品牌标识**
- **已装备装饰**：若已装备头像框/徽章则展示对应标签
- **个人信息**：昵称、ID、身份（如「TapTap 玩赏家」）、个性介绍

### DIY 编辑

编辑弹窗顶部为名片实时预览区，下方为三个 Tab 切换面板。Tab 顺序为：角色 → 背景 → 贴纸，默认展示角色 Tab。

#### 角色 Tab（默认展示）

- 展示所有角色选项（包括 Tap 头像、品牌形象角色、游戏角色等）
- 互斥选择，选中即应用
- **所有角色默认拥有，无需积分兑换**，点击即可直接切换
- 角色类型包括：图片角色（展示实际图片）、游戏角色（展示占位文字「游戏角色」）

#### 背景 Tab

- 展示所有背景主题选项（多种渐变色主题）
- 互斥选择，选中即应用
- 默认解锁 1 种基础背景，其余需消耗积分解锁
- 解锁后可自由切换

#### 贴纸 Tab

- 展示所有贴纸选项
- 可多选，上限 10 个
- 同种贴纸在名片上只能使用一次
- 点击已使用的贴纸 → 从名片上移除
- 点击未使用的贴纸 → 添加到名片（默认位置为右上区域）
- 默认解锁 1 种基础贴纸，其余需消耗积分解锁
- 已添加的贴纸支持拖拽调整位置，位置数据持久化

| 条件 | 表现 |
|------|------|
| 若贴纸已达上限，尝试添加 | 则 toast 提示「贴纸太多啦，先调整一下再添加」 |

#### 解锁流程（仅适用于背景、贴纸）

> 角色全部免费拥有，点击即可直接切换，无需解锁。

1. 用户点击未解锁的背景/贴纸选项
2. 弹出确认弹窗：展示名称 + 消耗积分 + 当前积分余额
3. 若积分足够 → 点击确认 → 扣除积分 → 解锁并立即应用 → toast「已解锁并应用」
4. 若积分不足 → 弹窗提示「积分不足，去游乐场/回顾领奖赚积分吧」

### 贴纸拖拽交互

- 使用 Pointer Events 实现拖拽
- 拖拽范围限制在名片区域内
- 点击贴纸可切换为「选中」状态
- 拖拽过程中实时更新贴纸位置（不触发完整 re-render，直接修改 style）
- 拖拽结束后持久化位置数据
- 贴纸位置数据结构：`{ id, x, y, s, r }`（id、x 百分比、y 百分比、缩放、旋转）

### 保存为图片

名片编辑弹窗 header 提供「保存为图片」按钮，点击后打开保存图片弹窗：

1. Canvas 实时渲染名片预览
2. 可勾选显示内容（背景、贴纸、角色）
3. 可选裁剪形状（方形/圆形）
4. 方形支持独立调节宽度和高度
5. 圆形支持调节直径
6. 裁剪框支持拖拽调整位置
7. 点击「保存图片」→ 按裁剪区域导出 PNG 并下载

### 分享名片

名片编辑弹窗 header 提供「分享」按钮，点击后打开分享弹窗：

- 展示当前名片预览（只读，贴纸不可拖拽）
- 展示分享链接二维码
- 提供「分享至」和「下载图片」按钮
- 下载的海报图包含：品牌标题、名片完整渲染（含背景、角色、贴纸、个人信息）、二维码区域
- 分享弹窗 header 提供「返回我的名片」按钮 → 关闭分享弹窗，重新打开名片编辑弹窗

### 分享链接进入

| 条件 | 表现 |
|------|------|
| 若通过 `#/sharememorial` 路由进入 | 则先渲染首页内容，然后自动打开名片分享弹窗（展示当前用户的名片），关闭弹窗后回到首页 |

## 模块七：十周年福利（积分商店）

### 入口

- 活动首页置顶区「十周年福利」入口 → 打开十周年福利弹窗
- 路由 `#/shop` → 跳转首页后打开十周年福利弹窗

弹窗顶部展示当前积分余额。

### 一、福利抽奖

#### 奖池

奖池包含 **5 个类别**，每个类别 2 个奖品：

| 类别 | 说明 |
|------|------|
| 头像框 | 获得后可在个人主页装备使用 |
| 点券 | 点券发放至账户余额，可购买站内游戏、CDKey、云玩服务等 |
| 优惠券 | 购买游戏时可用，有使用门槛和有效期 |
| 云玩时长 | 获得云玩兑换码，前往兑换中心兑换时长 |
| CDKey | 发放具体兑换码，前往兑换中心使用 |

每个奖品有独立的库存上限，抽完即止。

#### 抽奖流程

1. 每日限参与 1 次，消耗积分
2. 点击「参与抽奖」→ 奖池高亮轮播动画 → 落定到中奖奖品
3. 弹出中奖结果弹窗，展示奖品名称与图标
4. 若未中奖（所有奖品库存耗尽），弹窗展示「很遗憾，没有抽到」

#### 中奖结果弹窗（按类别差异化）

| 奖品类别 | 弹窗内容 |
|----------|----------|
| 头像框 | 提示「恭喜获得头像框，可在个人主页使用」 |
| 点券 | 提示「可购买站内游戏、PC CDKey、云玩服务等」，底部增加「查看我的钱包」按钮 |
| 优惠券 | 展示详细兑换路径（TapTap 安卓客户端 - 我的游戏 - 兑换中心），提醒尽快兑换 |
| 云玩时长 | 提示获得云玩兑换码，展示兑换路径（TapTap 安卓客户端 - 兑换中心） |
| CDKey | 在弹窗内直接展示兑换码内容（等宽字体高亮），引导前往兑换中心使用 |

所有弹窗均提供「知道了」按钮关闭，不展示消耗积分信息。

#### 状态规则

| 条件 | 表现 |
|------|------|
| 若今日未抽且积分充足 | 则「参与抽奖」按钮可点击 |
| 若今日已抽 | 则显示「今日已参与」（disabled），提示次日刷新 |
| 若积分不足 | 则「参与抽奖」按钮 disabled |
| 若奖池所有奖品均已抽空 | 则显示「奖池已抽空」（disabled） |
| 奖池中已抽完的单个奖品 | 置灰并标记「已抽完」 |

### 二、我的奖品

- 位置：十周年福利弹窗内，**抽奖区与兑换区之间**，以可折叠抽屉形式展示（仅在有中奖记录时显示）
- 默认收起，点击「我的奖品」标题栏可展开/收起
- 展开后以列表形式展示所有中奖记录，每条记录包含：奖品图标、奖品名称、获奖时间
- 按时间倒序排列（最新在最前）
- 不同类别的奖品在列表中展示额外信息：

| 奖品类别 | 列表中的额外内容 |
|----------|-----------------|
| 头像框 | 提示「可在个人主页使用」 |
| 点券 | 名称右侧显示「查看我的钱包」按钮（点击跳转钱包页） |
| 优惠券 | 提示「前往 TapTap 兑换中心查看」 |
| 云玩时长 | 提示「前往 TapTap 兑换中心使用」 |
| CDKey | 提示「前往 TapTap 兑换中心使用」 |

- 兑换区兑换的商品（头像框、徽章）**不**出现在我的奖品中

### 三、测试选项

抽奖区域底部提供测试工具栏（仅 demo 使用）：

- 「指定必中类型」下拉选择框，可选：随机（默认）/ 各奖品类别
- 选择后，下次抽奖将从对应类别的可用奖品中随机抽取（若该类别已抽空，回退为全池随机）
- 该选项不持久化，刷新页面后重置为「随机」
- 「重置今日机会」按钮：清除今日已抽状态，允许重新抽奖（今日未抽时按钮置灰）

### 四、积分兑换

兑换区位于我的奖品下方（两区域间有分割线），展示可兑换的纪念装饰商品（头像框、徽章等）。

#### 兑换流程

1. 用户点击商品的「兑换」按钮
2. 弹出确认弹窗：商品名 + 消耗积分 + 当前余额
3. 若积分足够 → 确认后扣除积分 → 商品进入用户资产 → 自动装备 → toast「已兑换」
4. 若积分不足 → 弹窗提示「积分不足」

#### 装备规则

| 条件 | 表现 |
|------|------|
| 若商品未拥有 | 则显示「N 积分兑换」按钮 |
| 若商品已拥有且已装备 | 则显示「已装备」（非高亮） |
| 若商品已拥有但未装备 | 则显示「装备」按钮（高亮），点击切换装备状态 |

装备后效果：

- 头像框/徽章标签展示在名片顶部右侧
- 名片 DIY 弹窗中同样展示对应装备按钮

## 模块间关系

### 积分流转

```
每日签到         ─┐
十年回顾卡片领奖 ─┤
绑定 Steam/角色  ─┤
发现好游戏领奖   ─┼── 积分 ──┬── 名片 DIY 解锁装饰（背景/贴纸，角色免费）
游乐场时长任务   ─┘          ├── 积分兑换（头像框/徽章）
                             └── 福利抽奖（头像框/点券/优惠券/云玩时长/CDKey）
```

### 导航关系

```
开场页 ──→ 活动首页 ←──→ 名片编辑弹窗 ←──→ 名片分享弹窗
                  │                          ←──→ 保存为图片弹窗
                  ├──→ 十周年福利弹窗（抽奖 + 我的奖品 + 兑换）
                  ├──→ 十年回顾分享弹窗
                  ├──→ 游戏故事弹窗
                  ├──→ 游戏详情说明弹窗
                  ├──→ 绑定 Steam 弹窗
                  ├──→ 绑定角色弹窗
                  └──→ 分享回顾页（#/sharerecap）
```

### 登录态影响

未登录用户始终停留在开场页，无法进入活动首页及任何功能模块。点击开场页进入按钮后弹出登录弹窗，登录成功后方可继续。

### 弹窗层级

活动使用单一弹窗容器，同一时间仅展示一个弹窗。弹窗之间的切换通过回调栈管理，关闭当前弹窗后可自动恢复上一层弹窗。

| 操作 | 弹窗切换 |
|------|----------|
| 名片编辑 → 分享 | 关闭编辑，打开分享弹窗；分享弹窗提供「返回我的名片」按钮 |
| 名片编辑 → 保存为图片 | 关闭编辑，打开保存弹窗；保存弹窗提供「返回名片」按钮 |
| 名片 DIY 点击未解锁项 | 打开解锁确认弹窗；确认/取消后回到名片编辑 |
| 十周年福利 → 兑换 | 打开兑换确认弹窗；确认后回到福利弹窗 |
| 十周年福利 → 抽奖结果 | 弹出结果弹窗；点击「知道了」后回到福利弹窗 |
| 十周年福利 → 我的奖品 | 页面内展开/收起抽屉（无弹窗切换） |

### 数据持久化

所有用户状态通过 `localStorage` 持久化。包含：

- 登录状态、开场/首次回顾标记
- 积分余额
- 已领取奖励 ID 列表
- 名片配置（背景、贴纸列表及位置、角色）
- 名片解锁列表
- 装备状态
- 游乐场游玩时长、已领取的时长任务档位
- 发现好游戏揭示/领取状态
- 生涯数据快照
- 福利抽奖中奖记录（含奖品 ID、类别、名称、图标、获奖时间）
- 每日签到/抽奖记录

以下划线 `_` 开头的临时状态（如测试选项、上次展示的游戏列表）**不持久化**，刷新页面后重置。

状态加载时进行字段合并与兼容处理，缺失字段使用默认值填充，异常数据回退到 fallback。
