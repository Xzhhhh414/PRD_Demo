# develop

从 PRD 和 Figma 设计稿出发，经过分析-编码-审查-测试的完整开发工作流。

## 调用方式

```
/prd-workflow:develop docs/xxx-prd.md
/prd-workflow:develop 功能关键词
```

## 执行流程概览

```
┌─ 需求输入 ────────────────────────┐
│ 定位 PRD → ★ 用户确认              │
│ Figma MCP 读取设计稿               │
└────────────────────────────────────┘
                  │
                  ▼
┌─ 需求分析 ────────────────────────┐
│ Developer Agent 分析实现方案        │
│ ★ 汇总疑问 → 用户确认              │
└────────────────────────────────────┘
                  │
                  ▼
┌─ 生成开发文档 ────────────────────┐
│ dev-log/[类型]-功能名-时间.md      │
│ ★ 用户确认后开始实施               │
└────────────────────────────────────┘
                  │
                  ▼
┌─ 编码实施 ────────────────────────┐
│ Developer Agent 写代码              │
└────────────────────────────────────┘
                  │
                  ▼
┌─ Code Review ────────────────────┐
│ Code Reviewer Agent 审查           │
│ ├─ PASS → 继续                     │
│ └─ FAIL → 修复循环（最多 3 轮）    │
└────────────────────────────────────┘
                  │
                  ▼
┌─ 测试 ───────────────────────────┐
│ Test Runner Agent                  │
│ 生成测试 → 执行 → 报告            │
└────────────────────────────────────┘
                  │
                  ▼
┌─ 用户验证 ────────────────────────┐
│ ★ 用户反馈 → 修复循环（如需）      │
└────────────────────────────────────┘
                  │
                  ▼
┌─ 自优化 ─────────────────────────┐
│ 采集信号 → ★ 提案确认 → 更新文件  │
│ 记录到 evolution.log              │
└────────────────────────────────────┘
```

## 产出物

- `dev-log/` 下的开发需求文档（含实施记录）
- 实际代码变更
- Code Review 报告
- 测试报告
- `evolution.log` 进化记录

## 涉及 Agent

- Developer (`agents/developer.md`)
- Code Reviewer (`agents/code-reviewer.md`)
- Test Runner (`agents/test-runner.md`)
