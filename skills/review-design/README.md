# review-design

通过 Figma MCP 读取设计稿，对照 PRD 审核覆盖度和交互完整性。

## 调用方式

```
/prd-workflow:review-design docs/xxx-prd.md
/prd-workflow:review-design docs/xxx-prd.md https://www.figma.com/file/xxxxx
```

## 执行流程概览

```
┌─ 定位 PRD ────────────────────────┐
│ 搜索/指定 PRD → 提取模块清单       │
└────────────────────────────────────┘
                  │
                  ▼
┌─ 获取设计稿 ──────────────────────┐
│ Figma MCP 读取文件结构             │
│ ★ 用户确认相关页面                 │
│ 下载页面截图                       │
└────────────────────────────────────┘
                  │
                  ▼
┌─ 设计审核 ────────────────────────┐
│ Design Reviewer Agent              │
│ 覆盖度 / 交互状态 / 可还原性       │
└────────────────────────────────────┘
                  │
                  ▼
┌─ 输出结果 ────────────────────────┐
│ PASS → 进入开发阶段                │
│ FAIL → 回 Figma 补充设计          │
└────────────────────────────────────┘
                  │
                  ▼
┌─ 自优化 ─────────────────────────┐
│ ★ 提案 → 用户确认 → 更新 Agent   │
└────────────────────────────────────┘
```

## 产出物

- 覆盖度矩阵（PRD 模块 vs 设计页面）
- 设计审核报告（PASS/FAIL + 问题清单）

## 涉及 Agent

- Design Reviewer (`agents/design-reviewer.md`)
