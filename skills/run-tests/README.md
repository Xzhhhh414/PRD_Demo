# run-tests

分析项目结构，生成测试用例，执行测试并报告结果。

## 调用方式

```
/prd-workflow:run-tests
/prd-workflow:run-tests src/utils/calculator.ts
/prd-workflow:run-tests 用户登录模块
```

## 执行流程概览

```
┌─ 环境分析 ────────────────────────┐
│ 识别测试框架、目录、配置            │
│ ★ 无框架时询问用户选择              │
└────────────────────────────────────┘
                  │
                  ▼
┌─ 确定范围 ────────────────────────┐
│ 指定文件 / git diff / dev-log     │
└────────────────────────────────────┘
                  │
                  ▼
┌─ 生成测试 ────────────────────────┐
│ Test Runner Agent                  │
│ 正向 + 反向 + 边界测试用例         │
└────────────────────────────────────┘
                  │
                  ▼
┌─ 执行测试 ────────────────────────┐
│ npm test / pytest / ...            │
└────────────────────────────────────┘
                  │
                  ▼
┌─ 输出报告 ────────────────────────┐
│ 通过率 / 失败详情 / 建议           │
│ ├─ 全部通过 → 完成                 │
│ └─ 有失败 → 分析原因 → 修正/告知   │
└────────────────────────────────────┘
```

## 产出物

- 测试文件（新建或更新）
- 测试报告（通过率、失败详情、建议）

## 涉及 Agent

- Test Runner (`agents/test-runner.md`)
